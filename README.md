# alvas_hack
# 🤖 Agentic CI/CD Pipeline Repair & Intelligent Release Orchestration

An AI-powered self-healing CI/CD system that automatically detects pipeline failures, diagnoses issues using Gemini AI, creates safe auto-fix pull requests, and orchestrates intelligent staging/production deployments.

---

# 🚀 Problem Statement

Modern CI/CD pipelines frequently fail because of:

- Missing dependencies
- Broken tests
- Configuration errors
- Timeout issues
- Deployment failures

Developers spend significant time debugging logs and manually fixing small issues.

This project introduces an **Agentic CI/CD system** that:

✅ Detects CI failures automatically  
✅ Uses AI to diagnose root causes  
✅ Creates automated pull requests for safe fixes  
✅ Re-runs CI pipelines automatically  
✅ Orchestrates intelligent deployments

---

# 🧠 What Makes This “Agentic”?

Traditional CI/CD:

```text
CI Fails → Human Fixes → Re-run Pipeline
```

Our system:

```text
CI Fails

↓

↓

↓

↓

↓

```

The AI behaves like an autonomous DevOps engineer.

# 🔄 Workflow Overview

## 1️⃣ CI Failure Detection

GitHub Actions runs tests automatically on every push.

Example:

```yaml
- name: Run Tests
  run: python -m pytest
```

If the pipeline fails, GitHub sends a webhook event to FastAPI.

---

## 2️⃣ Webhook Event Collection

FastAPI receives GitHub events:

```python
@app.post("/webhook/github")
async def github_webhook(request: Request):
```

Events are stored in:

```text
data/events.json
```

---

## 3️⃣ Gemini AI Diagnosis

Gemini analyzes:

- CI logs
- Failed tests
- Code diff

Example AI output:

```json
{
  "issue_type": "missing_dependency",
  "file_to_change": "requirements.txt",
  "change": "requests==2.32.0",
  "risk": "low"
}
```

---

## 4️⃣ Automated GitHub PR Creation

Using PyGithub:

- Create new branch
- Modify files
- Commit changes
- Open Pull Request

Example:

```python
repo.create_pull(
    title="Auto-fix missing dependency",
    body="Generated by Gemini AI"
)
```

---

## 5️⃣ Intelligent Release Orchestration

### ✅ Staging Deployment

Automatically deploys after successful CI.

### 🔒 Production Deployment

Requires manual approval using GitHub Environments.

This ensures human-in-the-loop safety.

# 🧪 Demo Flow

## Step 1 — Break CI

Remove a dependency from `requirements.txt`.

Example:

```text
ModuleNotFoundError: requests
```

---

## Step 2 — Show Webhook Event

GitHub sends failure event to:

```text
/webhook/github
```

Event appears in:

```text
data/events.json
```

---

## Step 3 — Trigger AI Agent

Call:

```text
POST /trigger/agent
```

Gemini analyzes logs and suggests fix.

---

## Step 4 — Apply Auto-Fix

Call:

```text
POST /apply/fix
```

System automatically creates GitHub PR.

---

## Step 5 — CI Passes

GitHub Actions reruns successfully.

---

## Step 6 — Staging Deployment

Staging deploy triggers automatically.

---

## Step 7 — Production Approval

Production deployment requires manual approval.

# 🛠️ Setup Instructions

## 1️⃣ Clone Repository

```bash
git clone https://github.com/YOUR_USERNAME/agentic-cicd-hack.git
cd agentic-cicd-hack
```

---

## 2️⃣ Create Virtual Environment

```bash
python -m venv venv
```

Activate environment:

### Windows

```bash
venv\Scripts\activate
```

### Linux / Mac

```bash
source venv/bin/activate
```

---

## 3️⃣ Install Dependencies

```bash
pip install -r requirements.txt
```
